# Design Direction

## Overall direction
- روشن
- مینیمال
- جدی
- آرام
- B2B premium

This site should feel:
- calm
- trustworthy
- structured
- product-led
- commercially serious

This site should not feel:
- generic SaaS
- flashy
- overly animated
- construction-cliche
- dark and aggressive

## Visual intent
PersianSaze should feel like a serious operating system for selling to active construction projects, not a generic template, directory, or marketing gimmick.

The visual system should communicate:
- clarity
- trust
- structure
- execution
- discipline

## Color direction

### Base colors
- Background main: warm off-white
- Surface/card: pure white
- Border: very soft gray
- Primary text: deep charcoal
- Secondary text: medium gray

### Accent color
- Primary accent: deep olive green
- Hover accent: slightly darker olive
- Soft accent background: very light olive tint

## Color behavior rules
- Use only one primary accent color across the homepage
- Primary CTA should use the accent color
- Secondary CTA should be outline or ghost
- Avoid multiple competing accent colors
- Avoid bright blue SaaS defaults
- Avoid orange construction clichés
- Avoid strong gradients

## UI rules
- Remove all English section labels such as:
  - Problem
  - Solution
  - System
  - Steps
  - Audience
- Borders should be soft and minimal
- Shadows should be minimal or nearly absent
- Corners should be moderately rounded, not overly round
- Use icons sparingly
- Prefer clean spacing over decorative elements
- Cards should feel calm and structured, not flashy
- Keep the interface light and breathable
- Keep section layouts simple and scan-friendly

## Typography direction
The typography should feel:
- clear
- confident
- restrained
- readable in Persian
- premium but not luxury-styled

### Typography rules
- Hero title should be strong and clear
- Section titles should be concise
- Paragraph width should remain readable
- Persian text should have comfortable line-height
- Do not overuse bold text
- Use hierarchy through spacing and scale, not visual noise

## Things to avoid
- generic template energy
- excessive gradients
- excessive animations
- heavy shadows
- too many visual motifs
- loud startup styling
- crowded cards
- overly decorative construction imagery
- English UI labels in the Persian homepage

## Implementation priority
When applying this design direction, prioritize:
1. color cleanup
2. CTA consistency
3. removal of generic labels
4. spacing and layout calmness
5. typography hierarchy
6. card simplification

## Execution note
Any visual refinement should improve trust, clarity, and structure without rewriting the locked Persian source copy.
