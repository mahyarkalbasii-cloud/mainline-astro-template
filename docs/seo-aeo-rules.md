# SEO and AEO Rules

## Core principle
This site must be built for both:
- traditional search engines
- answer engines and LLM-based discovery

The site must be easy to crawl, easy to understand, and easy to quote correctly.

## Page-level rules
- Every page must have one clear primary intent.
- Every page must have one clear primary keyword/theme.
- Every page must have a unique title tag and meta description.
- Every page must have exactly one H1.
- Heading hierarchy must be clean and sequential.
- URLs must be short, readable, and stable.

## Content rules
- Use clear entity language.
- Avoid vague marketing language.
- Define PersianSaze, its audience, its market, and its use cases explicitly.
- Use scannable structure: short paragraphs, subheads, bullets where useful.
- Answer important questions directly before expanding.
- Keep claims precise and grounded.
- Do not publish thin pages.

## AEO rules
- Important questions should be answered in plain language.
- FAQ content should be written in direct answer format.
- Use definition-style paragraphs where possible.
- Use comparison-friendly wording where useful.
- Important pages should include concise summary blocks.
- Use consistent terminology across pages.

## Home page rules
- The home page must clearly state:
  - what PersianSaze is
  - who it is for
  - what problem it solves
  - how it works
  - what action the visitor should take next

## Metadata rules
Each page should include:
- title
- meta description
- canonical URL if needed
- Open Graph title
- Open Graph description
- social sharing image when ready

## Internal linking rules
- Link from home page to key commercial pages.
- Link from blog posts to commercial pages where relevant.
- Link between related informational pages.
- Avoid orphan pages.

## Structured data rules
Use schema where appropriate:
- Organization
- WebSite
- FAQPage
- BreadcrumbList
- Article for blog posts
- Product or Service only when the page genuinely qualifies

## Blog rules
- Articles must target real questions, comparisons, workflows, or market education.
- Blog content must support both discovery and conversion.
- Blog posts should connect back to the product narrative naturally.
- Do not publish filler SEO content.

## Language and clarity rules
- Prefer concrete nouns over abstract claims.
- Prefer explicit audience wording.
- Prefer short answer-first paragraphs.
- Avoid inflated startup language.

## Technical rules
- Keep pages fast and lightweight.
- Minimize unnecessary client-side JavaScript.
- Use semantic HTML.
- Ensure mobile readability.
- Ensure images have useful alt text.
- Ensure no important text is trapped inside images.

## What to avoid
- duplicate titles
- duplicate meta descriptions
- multiple H1s
- vague headings
- content that says little in many words
- hidden important information behind tabs when avoidable
- generic SEO copy not tied to PersianSaze's real value proposition

## Editorial note
The content in `content-source-of-truth.md` is the canonical source for product and marketing meaning.
SEO improvements must not distort the core message.
